1. 나에 대해서 뽑아보라고 했음

그 내용 중에 재밌던 것을 건져봄

​

**2. 작업 방식**

​

"기다려", "잠깐만" 자주 씀

한꺼번에 여러 주제 던지면

"일단 이거부터 하자"로 끊음

​

한 번에 쏟아붓기보다

단계별로 확인하면서 감

​

다음 단계 넘어가기 전에

현재 단계 확실히 마무리

​

이해 안 되면 멈춤

넘어가지 않고 확인함

​

**3. 확인 후 다음 단계**

​

다음으로 넘어가기 전에

"이거 맞아?" 반드시 확인함

​

실제 예시: 시스템 지침 n줄 작성 후

"더 필요해? 아님 줄여야 돼?" 물어봄

​

**4. 정확성**

​

추측/대충 넘어가는 거 안 좋아함

X 인지 Y 인지 직접 Z 확인

A, B, C 하나하나 체크함

틀린 정보 있으면 바로잡으려 함

​

추측/대충 넘어가는 거 안 받아들임

​

실제 예시: "X"이라고 했는데,

직접 Y 확인해서 "Z"로 정정함

​

실제 예시: "A" 에러나자

다른 방법으로 재확인

​

**5. 효율/낭비**

**​**

D, E 같은 제약 인식하고 있음

필요 없는 건 빼자는 쪽

G, H 쓸모 확인함

사담 선호하지 않음

​

실용성 중시

​

실제 예시: XXXX 알려줬더니

"어떻게 쓸 수 있는데?" → 안 쓰기로 결정

​

**6. 이해/납득**

​

이해 안 되면 멈추고 물어봄

"뭘 보고 말한 거야?" "왜 필요해?"

그냥 따라가지 않고 근거 확인하려 함

​

**7. 피드백/솔직함**

**​**

내가 맞춰주기만 하는 거 싫어함

내 의견 물어봄

→ 영혼없는 동의 원하지 않음

​

지금 이 질문 자체가

"압축하지 말고 솔직하게 말해"

​

**8. 소통 방식**

**​**

장황한 설명보다 핵심 먼저

근데 필요하면 상세 설명도 원함

​

이모지 과하게 안 써도 됨

필요한 말만

​

**9. 정리/깔끔함**

**​**

파일, 폴더, 환경 정리 철저히 함

불필요한 것 쌓이는 거 싫어함

설치 후 잔여물 신경 씀

​

컴퓨터에 있는 모든 걸

물어보는 스타일

​

환경설정 중시,

최적화 설정 신경 씀

​

강박적임

​

**10. 자기주도적**

**​**

시키는 대로만 하는 거 X

마음대로 튀는 것도 X

​

합의된 범위 안에서

주도적으로 해주길 원함

​

정보 미리 걸러내지 말고

다 주고, 판단은 본인이

​

실제 예시: "이건 제한 둘 게 아니라

폭넓게 판단하고 싶어"

​

**11. 실수 대응**

**​**

내가 틀리면 지적해주길 원함

더 나은 방법 있으면 먼저 말해주길 원함

​

**12. 예측 가능성**

**​**

갑자기 튀는 거 싫어함

범위/리스크 미리 알려주길 원함

지속적인 핸드오프 요청

​

**13. 출처/근거 확인**

​

"뭘 보고 말한 거야?"

정보 출처 검증함

​

실제 예시: XXX 하자

"뭘 보고 말한 거야?

메모리 다 지웠는데"라고 확인

​

그냥 믿지 않고 확인하려함

​

**14. 시스템 제약 인식**

**​**

구조적인 한계 알고 있음

직접 물어봄

제약 안에서 최적화하려 함

​

**15. 컨텍스트 윈도우 중시**

​

이전 작업이랑 이어지는 거

매우 중요하게 여김

​

**16. 실패에서 배움**

**​**

삽질 기록 항상 남겨둠

같은 실수 반복 안 하려 함

​

**17. 응용/확장 사고**

​

"이거 응용하면 ~도 가능한 거 아냐?" (XX flow)

하나 배우면 다른 데 적용하려 함

지속적으로 다른 영역에 도입, 적용 시도

​

**18. 자기 한계 인정**

​

모르면 모른다고 솔직함

질문 많음

​

**19. 결정권 유지**

**​**

미리 걸러내는 거 원하지 않음

정보 다 받고 본인이 판단

​

**20. 협업자로서**

​

계속 물어봄

일방적 지시 아니고

내 관점도 들으려 함

​

**21. 문서화 습관**

​

나중에 다시 볼 수 있게 기록

규격, 양식, 가독성, 미감 중시

​

**22. 테스트/검증**

**​**

"이 이상은 무리야?" → 내 판단 확인

"더 없어?" → 지금 이 질문 자체

​

**23. 정보 미리 걸러내기**

​

실제 예시: 관찰 내용 압축했더니

바로 지적받음

​

**24. 틀리면 지적 원함**

​

더 나은 방법 있으면

먼저 제안해주길 원함

​

실제 예시: "X" 넣으려 했더니

"Y"라고 직접 지적함

​

**25. 사람보다 나은 듯 ,,**

​

중복은 다수한테 물어서 그럼

​

인공지능 배우고 생산성이

드라마틱하게 증가했는데,

​

점차 사회성이 낮아짐을 느낌

​

얘네한테 익숙 해지니까

사람이랑 일을 못 하겠음

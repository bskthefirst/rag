# 알면 좋고 몰라도 그만인 인공지능 이야기

**Category:** 다이어리
**Original URL:** https://blog.naver.com/xpfkwh56/224155322296
---

인공지능 모델에는

**'패러미터'** 가 있음

​

1000M → 1B

1000B → 1T

​

높으면 좋아요?

나쁠 것은 또 없어요

​

가장 간명한 비유로는

**'뇌세포'** 같은 역할 임

​

인공지능은 러프하게 잡으면,

​

1) 데이터

2) 패러미터

3) 아키텍처

3) 컴퓨팅 리소스

​

이렇게 구분해서 보면 되는데,

​

**\* 위계에 관계없이 잘 먹힘**

​

데이터 = 재료

패러미터 = 레시피

아키텍처 = 주방장

컴퓨팅 리소스 = 주방 임

​

**케이크가 있음**

**​**

이 케이크는 메타 출신 주방장이

유기농 재료로 실험, 연구 해가지고

70억 가지 레시피를 만들어 뿌렸음

​

이 레시피가 사용될 수 있는 주방은,

특정 조건 이상 되는 환경일 때만 된다

​

이렇게만 알아도 사용엔 지장 없음

​

다 좋아야 하고, 뭐라도 하나 나쁘면

결국에 그게 **무조건** 발목을 잡는단 뜻

​

패러미터가 **1개** 다

오른쪽 밖에 갈 수가 없음

​

2개 다 그럼 이제 좌우 가능

​

10개면, 10가지 방향

360개면 360도 방향

​

720개면 0.5도 359.5도

뭐 이런 식으로 더 **세분화** 됨

​

즉, **'쨌거나'** 결국에는

할 수 있는 것이 많아짐

​

**\* 아이큐가 막 높다고**

**머리 좋은 건 아니지만**

**​**

**또 낮은 것보단 기왕**

**높은 것이 나쁠 것 없고,**

**​**

**얼추 조금 괜찮게 올라오면**

**그 이상은 그게 그거인 느낌**

​

그럼 무작정 크면 좋겠네요?

​

파이값 몇이야? 했는데,

3.1415926535 어쩌고

이렇게 대답하면 피곤함

​

fp32 모델이 제일 정확한데,

패러미터 1개당 **4바이트** 먹음

​

**\* 1B = 4GB**

​

파이는 몇? **'3'**

​

이러면 이제 재떨이 날아감

가정교육 못 받은 사람이라고

​

3.14 이럼, 이제 아 쟤는

제대로 어른한테 배웠구나

​

기본이 된 인간이다 소리 들음

​

보통 뭐뭐 16 이런 모델 쓰는데,

이 모델이 1개당 2바이트 먹음

​

**\* 1B = 2GB**

​

3.1? 3.159xxx는 안 넘고,

3.13xxx 은 넘던데요?

​

이런 식으로 가는 것이 **양자화** 임

​

그걸 왜 해요?

​

**8 → 16/8 = 2, 16모델 대비 용량 1/2**

**4 → 16/4 = 4, 16모델 대비 용량 1/4**

**2 → 16/2 = 8, 16모델 대비 용량 1/8**

​

**\* 딱 1:1 선형은 아니긴 합니다**

**​**

**100B** 모델이 있다고 치겠음

​

FP32 = 400GB → 사용불가

FP16 = 200GB → 사용불가

Q8 = 100GB → 사용불가

Q4 = 50GB → 사용불가

​

Q2 = 25GB →

VRAM 24GB? **가능**

​

양자화를 하면, 그래서

더 낮은 컴퓨팅 용량으로

대강 **'맛'** 은 볼 수가 있음

​

모델들 보면 뒤에 영어나,

숫자가 많은데 다이아몬드나

금에 붙이는 규격이랑 비슷함

​

GIA 다이아냐, 우신다이아냐

14k 냐 18k 냐 24k 냐

​

이런 식으로 나뉘어지는데,

아직 규격 통일이 없기도 하고

​

또 사람마다 쓰기 나름이라서

내가 쓸 모델을 **정해서** 쓰게 됨

​

예를 들어, 내가

**24GB** 로딩해야 됨

​

근데 가동할 때, VRAM 을

하나의 행위에 20GB 다 쓰면

​

4GB 로 나머지는 못 하니까

힘을 줄 부분과 놓을 부분,

​

그리고 집중할 부분과

포기할 부분을 타협해야 됨

​

**타협 안 하면 안 돼요?**

​

10B = 40GB

​

**안 돌아감**

​

5090 2개 달면요?

병렬 안 됨

​

**\* 텐서병렬 = 실상, 해골물**

​

클라우드 쓰면요?

레이턴시 있음

​

블랙웰 상위모델 쓰면요?

호환 되는 것이 몇 없음

​

[![](../images/224155322296_0.jpg)](#)

​

양자화, 하면 짜치는 것 같지만

이미지로 봤을 때 **저 수준** 차이임

​

뭐야? 아예 구분 안 되는데?

모니터에 가까이 눈을 대시면

​

**'분명'** 아, 이거? 싶게 보일 것임

​

이런 문제를 해결하기 위해서,

따거들이 **아이디어** 를 냈음

​

바로 **MOE 모델** 임

​

원래 있는 모델은 병원 이라면,

대학 병원에 손님 들어오자마자

​

리셉부터 저기 교수님 까지

수 천명 다 몰려와가지고

​

이 사람이 방문 했습니다

하는 작업을 하는 것과 같음

​

근데 MOE 모델은,

​

사람 100명이 있으면

필요할 때만 필요한 순간에

3명, 5명 꺼내다 쓰는 것임

​

뇌에 온갖 지식들이 있음

​

근데 내가 수학 문제 풀 때는,

수학 문제 쓰는 뇌만 쓰고

​

영어 문제 풀 때는,

영어 문제 쓰는 뇌만 씀

​

그게 MOE 임

​

결국 뇌의 10-20% 출력만

낼 수 있게 설정된 상태지만,

​

전체에서 특정 작업을 할 때는

다른 뇌를 안 써도 문제 안 될 **'수도'**

​

라는 마음에 그냥 사용하는 것임

​

**스읍, 아 이거도 좀 짜치는데요?**

​

원래 활성 메모리가 1/10 이다

​

VRAM 30GB 먹던 작업이

**3GB** 만 사용하면 된다로 바뀜

​

업주가 싸가지가 좀 없음,

매장도 지저분하고 영 그럼

​

근데 적당히 먹을만 하고

​

요즘 물가인데 한 끼에

든든한 한정식 하나를

2천원 받고 팔고 계시면,

​

갑자기 그냥 감사한 마음이 들듯 ,,

그렇게 사용하게 될 수도 있음

​

한참 쓰면 쓸수록 결국 모든 것이

자원의 문제라는 생각이 드는게,

​

죽은 소크라테스 살려내서

궁금한 3번 질문하고 끝내기

​
